# PokerGame 美术方向

## 文档目的

本文把当前已经生成并接入运行时的视觉方向整理成后续 UI 和美术修改必须遵守的基线。它描述风格、色板、像素表现、动态内容边界和资产维护方式，不代替具体资源清单。

本文仅根据当前仓库中的 `scripts/ui/main.gd`、`assets/art/generated/style/style-guide.png`、生成 Prompt 和已接入资源整理；未在仓库中确认的字体授权、动画完成度和发布平台要求均标记为 `待确认`。

## 当前视觉定位

- 主题：原创的“深夜地下牌局”，带有温暖黄铜灯光和深绿色牌桌。
- 气质：战术、克制、略带冒险感，优先保证信息清晰；不走霓虹、奢华赌场或夸张街机路线。
- 表现：现代独立游戏式像素美术，使用清晰剪影、硬边、有限色板和少量强调色。
- 视角：牌桌以接近正交的轻俯视构图为主，动态 UI 叠加区域应保持安静和低细节。

## 核心色板

| 用途 | 色值 |
| --- | --- |
| 近黑背景 | `#050B0A` |
| 深色面板 | `#0E110F` |
| 深绿牌桌 | `#0F2B21` |
| 黄铜强调 | `#CBA14D` |
| 米白文字与牌面 | `#EDDFB8` |
| 蓝灰次级操作 | `#32505D` |
| 暗红危险状态 | `#9E2E2A` |

新增资产应优先使用这七种核心颜色。若图像生成或后处理产生近似色，应在运行时接入前做色板归一。

## 像素与材质规则

- 使用真实硬边像素；轮廓通常为 1-2 像素。
- 使用 clipped corners、平坦色块和克制的单步高光表现层级。
- 禁止抗锯齿、渐变、柔化阴影、模糊、辉光、写实材质和 3D 渲染感。
- 运行时纹理保持 nearest-neighbor filtering，避免线性过滤造成模糊。
- UI atlas 必须使用固定网格、相同单元尺寸和透明间距；接入前复核切片区域与 nine-slice 边距。
- 牌桌、面板和装饰不能抢占公共牌、筹码、行动按钮或事件文本的可读空间。

## UI 层级

- 主要操作使用黄铜色，例如开始、确认和当前重点行动。
- 常规操作使用蓝灰色。
- 危险操作使用暗红色，例如全下或统计重置确认。
- 当前行动者、盲注和获胜状态需要强可见标识；已弃牌或淘汰状态使用降低亮度和饱和度表达。
- 动画和反馈应短促、像素化且不遮挡牌局信息。当前仓库虽包含 VFX 参考图集，但运行时动画完成度仍待确认。

## 动态内容边界

以下内容必须由 Godot 运行时绘制，不应固化进 PNG：

- 中文按钮和状态文字；
- 扑克牌点数与花色；
- 筹码、底池、跟注和加注数值；
- AI 名称、人格、行动和牌局事件；
- 胜负、牌型、统计与其他会变化的数据。

这样可以保证规则结果准确、中文内容可修改，并避免生成图片中的错误文字进入产品。

## 非目标

- 本文不要求一次性接入仓库中的全部图集。
- 不把生成图片中的中文字体样张直接作为可安装字体使用。
- 不为了视觉效果改变扑克规则、AI 决策或 `PokerRound.apply_action()` 的权威边界。
- 不在未确认发布平台前定义移动端、Steam 或高 DPI 的最终规格。

## 验收标准

- 新增或替换的视觉资产符合核心色板和硬边像素规则。
- 动态文字、牌值和游戏数值仍由运行时生成。
- UI 在 1280x720、1440x900 和 1920x1080 的布局探针中不越界。
- 关键交互状态可辨识，纹理没有线性过滤模糊。
- `tests/ui_layout_probe.gd` 与主场景启动检查通过。
